The Block Project Matchmaking App

To reduce the manual work on matchmaking the hosts and the residents who are experiencing the homelessness

Overview

This is my personal side-project in cooperation with the team of The Block Project which aims to invite the community into the task of ending homelessness by placing a BLOCK Home in the backyard of one single-family lot on every residentially zoned block in Seattle. Being inspired by the vision, I proactively proposed the matchmaking idea for the team to better the matching experience for those who want to do good. To start
Firstly, I’ve reached out to my friends from diverse backgrounds and would be interested in the project and I lead a team composed of members from:

  • Business marketer
  • Interactive Design
  • Full stack software engineer
  • Back-end engineer

For this project, I contributed as the Project Leader, Case Researcher, UI Designer, and Front-end Developer. As the project's team leader, my responsibilities included managing the project development from setting up the project working timeline to arranging the meeting with team members and the client. As the role of Case Researcher and UI Designer, I contributed collaboratively to the project's research phases, ideation and wireframe design. For the Front-end Developer, I’ve cooperated with a senior full stack engineer, Ben, to develop the HTML5, SCSS, and JavaScript.

Research

Learn from People:
To better understanding the project, I conducted the interview with the staffs from The Block Project to understand their current matchmaking process and the challenges and goals for the Block Project. And there are the key takeaways from the interview:

  • 4 User Types: Hosts, Residents, Caseworkers, and Administrators
  • Time Consuming: Long paperwork, tedious form, and lengthy matching process
  • Lack of Human Beauty: Little human beauty element exists during the matching process

Analogous Inspiration:
To learn how to design a user-friendly form for a matchmaking app, the Airbnb and HomeAway are chose to be our comparison examples as the former shares the strong connections between the host and the guest while the later are focusing on providing the best for the family’s memory.

Below are the 3 key design patterns that may help the users complete the form in a more seamless way:

1) The progress bar, which indicates how much the users have been done and how far needed to be completed, might help users to easing the uncertainty


2) Side notes and ideas may be a great helper for the users to facilitate the form


3) Clear clarification on what the question will be used gives the users a greater peace of mind

Ideation

We conducted the user flow which helped to know what challenges will the users encountered in each phase.

Afterward,we developed three concepts to reach the goal of optimizing the current matchmaking process in 2 phases.

Solution

The high-fidelity design of the user flow are created to create the matchmaking process which is more accessible, easy-to-use, and highlights the core value of nature of people

Reflection

Team management

  • This web app is much more ambitious than expected. As it is the first time I lead a web application development, I ignored a couple of details which I learned during the production.
  • One of the most important thing is the project flow- the UI goes first, the front-end development will be the second, the back-end and testing. All of three can not be conducted at the same time

UI design

  • My first time use sketch to come up with the wireframe and UI design from self-taught experience. It took a while to pick up the tools and realized the template would help scale up the whole project and reduced a bunch of time

Front-end development

  • As the senior developer suggested to use the Ember.js to build this complex system and will be easy to maintain in the near future, I could barely apply what I just learned from the basic javascript class to the project than I expected
  • However, I still studied the concept of the framework on my own and learned how to write simple javascript (such as name filtering and incremental buttons) with the help from the senior member
  • There are more than 45 page name on the site which is one of the troublesome things I encountered at first. It took me awhile to make them consistent. It would be helpful and more efficient if I come up with a list of the page names before creating the pages
