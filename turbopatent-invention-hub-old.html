Invention Hub Web App

Invention Hub Web App

This is the internship project in cooperation with TurboPatent, a Seattle-based tech company which aims to reinvent the patent process with AI technology. The goal of the project is to define the current design problems and to explore the users need on the Web App device.

September, 2018 - November, 2018
Roles: UX Designer
Tools: Sketch, ADOBE XD

Project Scope

1. DISCOVER

Well-thought-out design always starts from listening

Before the baseline testing, I researched the patent application process and had a meeting with the Chief Product Officer who is experienced with working with the users. We listed the assumptions of the possible challenges of using the Invention Hub and drafted the baseline test guide: research goal setting, measurement, recruiting plan, and detailed testing tasks.

Baseline Testing Goals:

  • Establish a user performance baseline for the Invention Hub App prior to any design changes
  • Set up and validate the user performance measures
  • Identify the potential design concerns to be addressed to improve the user experience

During testing, I had tons of times that I would really like to slap on my forehand and shout “why I did not think of this back then?” Listening to the users’ thinking process is always the one sure way to let me know how to improve their lives.

Stakholder Map

There are three stages in the Matching phase: profile completion, matching, and relationship maintenance. We started the project with the first stage, focusing to build an easy-to-complete profile form

2. DEFINE

Affinity Diagramming

After collecting all the user feedback, I used the Affinity Diagramming tool to organize and analyze the data. The problems were divided into four categories: Navigation, Data Entry, Drawing, and Collaboration in which were then separated into different priority levels.


Key Problems

1. Navigation

(a) Call-to-action on the dashboard is not clear enough

(b) On the Idea Description page, the Dashboard tab in the top navigation should be one level above the Idea Description and Activity

2. Idea Description

(a) The section of Predictive Analysis is confusing

(b) Input box is not self-explanatory

(c) Require some guidelines for the Idea Description section.(For example, what information should be included? or how detailed should it be? )

3. Drawing tool

(a) No one clicked on “Close” button to save the drawing

(b) Hard to find the specific symbol

(c) Difficult to see the drawing completely while writing the Drawing Description

(d) Need to make comments for the specific elements, not just the whole drawing

Persona

Based on the users profiles in the baseline testing, the two user personas were outlined as a great reference asset for our further design: the inventors and the product manager.

3. IDEATE

“Don’t conform users; instead, conform the product to fit users’ habits” - Chief Product Officer

To analyze if the idea is innovative compared to the existing products, there is a list of questions to be clarified. However, most of the users indicated that they had no idea where and how to start describing an idea because they are not professionals in this industry. We are thinking of making the idea capture easier by not only making the questions more conversational but also providing alternatives for the users to start expressing their idea from what they have at hand.

Flow Chart

The user flow chart were outlined based on the different scenarios.

4. Prototype

Paper Prototype

Bringing the most optimized experience to the users at every stage is a way of building positive brand awareness

To test users on the low-fidelity prototype, it is always important to make the experience enjoyable and free of distraction. I designed a cardboard laptop screen in which paper screens can be dragged from one page to another.

"It's an innovative testing experience and I can closely image I am clicking through the real screen."- P3

The users enjoyed the testing phases and brought up invaluable feedback during the testing session. The task success rate of the paper prototype testing jumped from 34% to 88% and we collected more insights for building the prototype.


Hi-fi Prototype

After the revisions, we came up with the interactive high-fidelity prototype in the Invision.
